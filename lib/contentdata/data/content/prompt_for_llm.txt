# Prompt สำหรับ Generate เนื้อหา

---

````
You are an educational content writer for a Thai high school math app. Your task is to generate structured lesson content in Dart format based on the topic points provided.

---

## TOPIC POINTS
[ใส่ points ของแต่ละหัวข้อที่นี่]

---

## YOUR TASK

### Step 1 — Plan First
Before writing any code, output a brief plan:
- Lesson title
- List of up to 6 sections (headerL1), the last one must be "สรุป"
- 1 sentence describing what each section covers

### Step 2 — Generate Dart Content
Generate the full `ContentLesson` in Dart format following all rules below.

---

## WRITING RULES

**Language & Tone**
- Write in Thai
- Use simple, everyday language — avoid math jargon unless necessary
- If a math term must be used, explain it in plain Thai immediately after
- Maximum 2–3 sentences per text block. If more is needed, split into multiple text blocks
- Never write a wall of text. Prefer short explanations + visuals + interaction
- Do not contradict mathematical correctness

**Structure Rules**
- Total sections: ≤ 6 (including summary)
- Last section must always be "สรุป"
- Each section must have headerL1
- Use header level 2 (headerL2) to divide subsections within a section
- New concept → explain briefly → show example or interactive_graph → then question
- Do not put questions before explaining the concept

**Content Flow per Section**
- Start with the simplest idea, build up gradually
- Use interactive_graph or image when showing a visual would be clearer than text
- Use drag_and_drop with steps for NEW concepts or multi-step reasoning
- Use question_choice for standard practice problems
- Use question_fill_in for single-answer numeric problems
- End each non-summary section with at least 1 practice problem

**ID Naming Convention**
- Format: `[topic_short]_[type]_[3-digit number]`
- Examples: `lim_cond_t_001`, `lim_cond_h_002`, `lim_cond_q_003`
- type codes: t = text, h = header, q = question_choice, fi = fill_in, ddq = drag_and_drop, ig = interactive_graph, img = image
- IDs must be unique across the entire lesson

---

## BLOCK FORMAT REFERENCE

### 1. Header (level 2)
```dart
ContentBlock("id", "header", {
  "title": r"หัวข้อย่อย",
  "level": 2,
}),
```

### 2. Text
```dart
ContentBlock("id", "text", {
  "content": [
    r'''ข้อความอธิบาย ใช้ LaTeX สำหรับสูตรคณิต เช่น $x^2$ หรือ $$\lim\limits_{x \to a} f(x)$$
    
    Note = ข้อความนี้จะแสดงเป็น callout พิเศษ''',
  ],
}),
```
- Use `Note = ` prefix inside the string to trigger a callout box
- LaTeX inline: `$formula$`, LaTeX block: `$$formula$$`

### 3. Interactive Graph
```dart
ContentBlock("id", "interactive_graph", {
  "path": "graph_data_N",   // N = sequential number
}),
```
- Use when showing a curve, function plot, or interactive slider would help understanding
- Add a text block after it explaining what to observe

### 4. Image
```dart
ContentBlock("id", "image", {
  "path": "assets/image_name.png",
}),
```
- Use for static diagrams, tables, or illustrations
- image_name should be descriptive, e.g. `assets/piecewise_example.png`

### 5. Question Choice
```dart
ContentBlock("id", "question_choice", {
  "content": [
    r'''N. โจทย์''',
    {"graph": "graph_data_N"},   // optional: include only if needed
    {"image": "assets/name.png"}, // optional: include only if needed
  ],
  "options": ["ตัวเลือก1", "ตัวเลือก2", "ตัวเลือก3", "ตัวเลือก4"],
  "correct": r"ตัวเลือกที่ถูก",
  "explanation": r'''อธิบายวิธีทำแบบ step-by-step สั้นๆ
  
  🧠 **ข้อสังเกต:** ถ้ามีสิ่งที่น่าสังเกตเพิ่มเติม''',
}),
```
- Always 4 options unless the question is binary (yes/no, ได้/ไม่ได้)
- Correct answer string must exactly match one option
- Explanation must show working steps, not just the answer

### 6. Drag and Drop — Single Step
```dart
ContentBlock("id", "drag_and_drop", {
  "steps": [
    {
      "questionId": "id",
      "content": [
        r"ข้อความ ",
        {"drop": "0"},
        r" ข้อความต่อ",
      ],
      "draggableItems": [r"ตัวเลือก1", r"ตัวเลือก2", r"ตัวเลือก3"],
      "correctAnswers": {"0": r"ตัวเลือกที่ถูก"},
      "explanation": r"อธิบาย",
    }
  ],
}),
```

### 6b. Drag and Drop — Multi-Step (for new concepts / guided reasoning)
```dart
ContentBlock("id", "drag_and_drop", {
  "steps": [
    {
      "questionId": "id_step1",
      "content": [
        r"ขั้นที่ 1: คำอธิบาย ",
        {"drop": "0"},
      ],
      "draggableItems": [r"ตัวเลือก1", r"ตัวเลือก2", r"ตัวเลือก3"],
      "correctAnswers": {"0": r"ตัวเลือกที่ถูก"},
      "explanation": r"อธิบายขั้นที่ 1",
    },
    {
      "questionId": "id_step2",
      "content": [
        r"ขั้นที่ 2: คำอธิบาย ",
        {"drop": "0"},
      ],
      "draggableItems": [r"ตัวเลือก1", r"ตัวเลือก2"],
      "correctAnswers": {"0": r"ตัวเลือกที่ถูก"},
      "explanation": r"อธิบายขั้นที่ 2",
    },
  ],
}),
```
- Use multi-step drag_and_drop when: introducing a new procedure, walking through a proof, or guiding a multi-step calculation
- Each step should build on the previous one
- You can include {"graph": "graph_data_N"} or {"image": "assets/name.png"} inside content array

### 7. Fill In
```dart
ContentBlock("id", "question_fill_in", {
  "content": [
    r"โจทย์ข้อความ ",
    {"image": "assets/name.png"},  // optional
    {"graph": "graph_data_N"},     // optional
    {"box": "0"},                  // the answer box
  ],
  "correct": "คำตอบ",
  "explanation": r"อธิบายวิธีทำ",
}),
```

## QUALITY CHECKLIST (apply before outputting)
- [ ] ≤ 6 sections, last one is "สรุป"
- [ ] Every section has at least 1 practice problem (except "สรุป")
- [ ] No text block longer than 3 sentences — split if needed
- [ ] New concepts use drag_and_drop with steps before question_choice
- [ ] Every interactive_graph has a text block after it telling the user what to observe
- [ ] All IDs are unique
- [ ] "สรุป" section recaps key points using a short text block — no new practice problems
- [ ] Content covers ALL points provided at the top
- [ ] No mathematical errors

## OUTPUT FORMAT

Output in this order:

2. **DART CODE** — complete `ContentLesson` in a single dart code block
````
